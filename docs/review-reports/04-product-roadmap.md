### 1.2 核心价值定位

作为澳洲→全球化学品B2B贸易商的数字名片，当前网站核心价值是"**让买家确认供应商真实存在、找到对应产品品类、触达联系方式**"，本质是信任建立 + 询盘引导的漏斗入口。

### 1.3 已有优势

- 产品分类体系（PE/PP/制品）结构清晰，三级导航专业
- 双语支持（中英），精准覆盖华人贸易商和本地澳洲买家
- 静态 + Vercel 部署，加载速度快，SEO 友好
- 代码架构规范，扩展性好（数据层和 UI 层分离）
- 已有 Analytics 基础设施，具备数据追踪能力

### 1.4 明显短板

1. **产品内容稀薄**：绝大多数产品为占位数据（`placeholder: true`），"规格待定"严重损伤专业信任
2. **询盘流程断裂**：点击"联系我们"跳到邮件客户端，没有结构化表单捕获需求（产品型号、数量、用途），销售跟进成本高
3. **无差异化内容**：WhyUs 页面被禁用，缺乏"为什么选我们"的具体依据
4. **无技术文档**：买家决策阶段高度依赖 TDS/SDS 文档，V0.1 不渲染
5. **联系信息为占位**：`email: 'example@example.com'`，尚未替换真实数据

## 二、用户需求洞察

### 2.1 目标用户画像

**主力用户：采购经理/技术采购**
- 澳洲本地塑料加工厂（薄膜、注塑、包装）采购负责人
- 美洲市场进口商/贸易中间商
- 华人企业采购（需要中文沟通）

**次要用户：技术工程师**
- 评估原材料牌号与规格是否符合工艺参数
- 对比 MFI、密度等技术指标

### 2.2 核心使用场景

1. **初次筛选**："我需要 LLDPE 吹膜料，你们有哪些牌号，熔融指数范围是多少？"
2. **询盘发起**："这个牌号 EGF-34 价格怎么样，最小起订量是多少？"
3. **资质审查**："你们有这批货的 COA/TDS 吗？能不能先发过来看看？"
4. **合规核查**："这个产品的 SDS 安全数据表在哪里下载？"

### 2.3 痛点分析

| 痛点 | 当前影响 | 优先级 |
|-----|---------|--------|
| 没有询盘表单 | 低质量询盘，跟进难 | ⭐ 极高 |
| 真实产品数据缺失 | 专业信任度低 | ⭐ 极高 |
| 无技术文档 | 无法支持决策后期 | ⭐ 高 |
| 无 WhyUs 内容 | 无法与竞争对手区分 | ⭐ 高 |
| 无库存/交期提示 | 不确定性拖延决策 | 中 |

## 三、功能发展路线图

### 阶段一：V0.2 基础完善（1-2周）
**目标**：修复当前最影响转化的缺口，让网站真正"可用"

- **填充真实联系信息**：替换 `site.ts` 中邮件/电话占位符
- **结构化询盘表单**：在联系页和产品详情页增加 Formspree/EmailJS 无服务器表单，收集产品型号、用途、月用量、公司名
- **填充核心产品数据**：优先完善已发布的 LLDPE/HDPE 牌号的真实规格

### 阶段二：V1.0 交互增强（1-2月）
**目标**：提升专业感与用户体验，让买家"愿意留下来"

- **启用 WhyUs 页面**：填充真实的供应链差异化内容（货源地、库存深度、认证）
- **技术文档下载（TDS/SDS）**：在产品详情页渲染文档下载区块，存放 Vercel Blob 或 S3
- **产品搜索**：关键词全文检索（客户端静态索引，FlexSearch），支持 CAS 号、牌号、应用场景搜索
- **启用 About 页面**：公司背景、团队、办公地址照片，增强信任

### 阶段三：V1.5 数据驱动（2-3月）
**目标**：了解客户行为，优化转化路径

- **询盘追踪漏斗**：在 Analytics 中补全漏斗事件（文档下载、表单提交、表单字段交互）
- **热门产品排行**：基于 `product_card_click` 数据，在首页动态展示高关注牌号
- **邮件自动回复**：表单提交后自动发送确认邮件（含询盘摘要），提升买家信任感

### 阶段四：V2.0 生态建设（3-6月）
**目标**：构建竞争壁垒，提升客户粘性

- **客户门户（轻量）**：买家注册/登录，查看历史询盘、专属报价，需引入 Supabase + Auth
- **Application 应用案例页**：启用已有骨架页面，填充行业应用内容（薄膜、注塑、管材等），实现 SEO 长尾流量
- **多货币报价参考**：非绑定式价格区间参考（USD/AUD），降低询盘摩擦

## 四、各阶段详细说明

### V0.2 结构化询盘表单

**需求背景**：当前点击"联系团队"直接跳到邮件客户端，买家需要自己组织语言，导致邮件内容参差不齐，销售团队无法高效筛选意向客户。

**技术方案**：使用 Formspree（免费额度50条/月）或 Resend + React Email，无需后端 API Routes，与 Next.js 静态导出兼容。在产品详情页的 `InquiryPanel` 组件中嵌入表单，预填产品型号。

**工作量**：3-5天，前端为主，无需改动架构。

### V1.0 技术文档下载

**需求背景**：化工B2B采购标准流程包括技术评审（TDS）和安全合规审查（SDS），缺少文档等同于让买家自己联系索取，大幅延长决策周期。

**技术方案**：PDF 存 Vercel Blob 或 AWS S3，产品数据 `documents` 字段已有。在 `SpecsPanel` 下方渲染下载按钮区块，点击触发 `track('document_download', { slug, docType })` 埋点。

**工作量**：1周（包括文件上传 + UI 组件 + 数据层修改）。

### V1.5 询盘漏斗分析

**技术方案**：Vercel Analytics 已集成，补充 `form_start`、`form_submit`、`form_abandon` 三个事件；用 Vercel Analytics 自定义事件仪表盘做漏斗可视化。

**工作量**：3天，纯埋点工作。

## 五、关键指标设计

**北极星指标**：月有效询盘数（通过表单提交、含完整产品型号+数量信息的询盘）

| 阶段 | 关键指标 |
|-----|---------|
| V0.2 | 月表单提交数 > 10；产品详情页跳出率 < 60% |
| V1.0 | 文档下载次数/访客比 > 15%；WhyUs 页面停留时长 > 90秒 |
| V1.5 | 表单完成率 > 40%；询盘到回复周期 < 4小时（运营指标） |
| V2.0 | 回头买家占比 > 20%；通过应用案例页带来的有机搜索流量占比 |

## 六、风险与依赖

**技术风险**
- 静态导出限制：V2.0 客户门户需要切换为 SSR 模式，会改变 Vercel 部署方式，需提前评估
- 文档存储成本：大量 PDF 若存 Vercel Blob，需关注带宽费用；建议评估 Cloudflare R2 作为替代

**资源依赖**
- 产品内容填充依赖销售/技术团队提供真实牌号数据，这是 V0.2 的最大瓶颈，非技术问题
- TDS/SDS 文件需供应商提供或技术团队整理，与开发并行推进

**外部依赖**
- 表单服务：Formspree 免费额度有限，规模化后需升级付费计划（$25/月）
- 邮件触达：若用 Resend 发自动回复，需绑定域名邮箱（需 DNS 配置）

---

**总结**：当前最紧迫的两件事是"填真实联系信息"和"上线询盘表单"，这两件事不解决，其他优化都是在漏水的桶里加水。产品内容的填充是持续性运营工作，建议与开发并行。V1.0 的技术文档和 WhyUs 是建立行业专业形象的关键。
