# 化工官网技术选型确认

## 1. 技术栈结论

本项目推荐采用以下技术栈：

| 模块 | 选择 | 说明 |
| --- | --- | --- |
| 前端框架 | Next.js App Router | 支持静态生成、SEO、后续 API 与后台扩展 |
| 开发语言 | TypeScript | 降低产品数据、页面参数、后台接口维护风险 |
| 样式方案 | Tailwind CSS + 少量自定义 CSS | 适合快速复刻现有设计，并保持组件化和一致性 |
| 一期内容管理 | 结构化数据文件，例如 `products.ts` / `products.json` | 快速上线，不依赖数据库，产品不直接写死在页面里 |
| 一期部署 | Vercel | 海外部署友好，Next.js 支持最好，接入域名简单 |
| DNS / CDN | 已购买域名 + 可选 Cloudflare | 用于域名解析、CDN、安全和缓存优化 |
| 二期数据库 | Supabase | 管理产品、分类、文件、询盘、AI 客服知识库 |
| 二期后台 | Next.js 管理后台 | 复用同一套技术栈，提供产品维护和询盘管理 |
| 二期 AI 客服 | OpenAI API + Supabase + Next.js API Routes | 基于产品资料和 FAQ 回答问题，并记录线索 |

## 2. 为什么选择 Next.js

### 相比纯静态 HTML

当前原型是 HTML / CSS / JS，适合设计展示和早期验证，但随着产品数量增加，会出现这些问题：

- 产品容易分散写在多个页面里，后续维护成本高。
- 新增产品时需要手动复制详情页，容易出现内容不一致。
- SEO 信息、产品详情、相关产品和首页推荐难以统一管理。
- 二期增加 AI 客服、询盘接口、后台时需要重新搭建服务端能力。

Next.js 可以保留静态站的速度和低成本，同时为后续数据库、后台和 AI 客服预留扩展空间。

### 相比 WordPress

WordPress 适合内容型网站，但本项目更像“产品展示 + 询盘转化 + 未来 AI 客服”的业务网站。选择 Next.js 更适合：

- 页面设计可控，方便还原当前视觉稿。
- 产品模型、对比模块、文档下载、AI 知识库更容易结构化。
- 性能和 SEO 更稳定。
- 二期可直接扩展 API、后台、AI 客服，不需要受插件生态限制。

## 3. 一期产品数据方案

一期不建议把产品直接写死在首页或产品页组件里，也不建议一开始就运行时查数据库。

推荐做法：

- 产品数据集中维护在结构化文件中，例如 `src/content/products.ts`。
- 首页只展示 `featured: true` 的产品。
- 产品列表页读取全部 `published: true` 的产品。
- 产品详情页根据产品 `slug` 静态生成。
- 产品分类、相关产品、对比信息都从同一份数据读取。
- 新增产品时只需要新增一条产品数据、准备图片和文件，然后重新部署。

建议第一期产品字段包括：

```ts
type Product = {
  slug: string;
  name: string;
  nameZh?: string;
  category: string;
  casNo?: string;
  summary: string;
  summaryZh?: string;
  grade?: string;
  purity?: string;
  packaging: string[];
  applications: string[];
  documents: Array<"COA" | "TDS" | "SDS">;
  image: string;
  featured: boolean;
  published: boolean;
  sortOrder: number;
};
```

## 4. 后续如何添加新产品

一期新增产品流程：

1. 准备产品基础信息：名称、分类、CAS、简介、规格、包装、应用、文件类型。
2. 准备产品图片和相关资料文件。
3. 在产品数据文件中新增一条产品记录。
4. 设置 `published: true` 控制是否展示。
5. 设置 `featured: true` 控制是否出现在首页。
6. 重新部署网站。

二期新增产品流程：

1. 管理员登录后台。
2. 在产品管理页面新增产品。
3. 上传图片、SDS、TDS、COA 等文件。
4. 设置分类、推荐状态、发布状态和排序。
5. 保存后触发前台更新。
6. AI 客服知识库同步更新已发布产品内容。

## 5. 一期 MVP 功能拆分

一期目标是快速上线、建立公司可信度、展示核心产品并收集询盘。

### 页面

- 首页
- 产品列表页
- 产品详情页
- Why Us 页面
- About 页面
- Contact 页面

### 核心模块

- 公司定位和首屏主视觉
- 首页推荐产品
- 产品分类
- 产品详情
- 产品竞品对比
- 相关产品推荐
- 行业应用入口
- 质量、文件、包装、供应能力说明
- 联系表单
- 邮箱、电话、WhatsApp、地址展示
- 中英文内容切换
- 基础 SEO 配置

### 暂不包含

- 公开报价
- 在线下单
- 在线支付
- 会员系统
- 复杂后台
- 实时库存

## 6. 二期功能拆分

二期目标是让运营人员可以维护产品，并增加 AI 客服和线索管理能力。

### 管理后台

- 管理员登录
- 产品新增、编辑、删除、上下架
- 产品分类管理
- 首页推荐产品管理
- SDS / TDS / COA 文件上传和管理
- 询盘列表
- 询盘状态管理
- AI 客服聊天记录查看
- 客户线索摘要

### AI 客服

- 网站右下角聊天入口
- 基于已发布产品、FAQ、产品资料回答问题
- 收集客户姓名、公司、邮箱、电话、目标产品和需求描述
- 自动生成询盘摘要
- 将敏感问题、不确定问题、价格问题、合规承诺问题转人工
- 后台查看聊天记录和人工介入状态

## 7. 二期数据库建议

推荐使用 Supabase，核心数据表可包括：

- `products`
- `categories`
- `product_documents`
- `inquiries`
- `chat_sessions`
- `chat_messages`
- `ai_knowledge_chunks`
- `admin_users`

第一期数据结构设计时应尽量贴近未来数据库字段，降低迁移成本。

## 8. SEO 与部署建议

一期每个产品详情页应生成独立 SEO 信息：

- 页面标题
- 页面描述
- 产品 slug URL
- Open Graph 图片
- canonical URL

部署建议：

- 使用 Vercel 部署 Next.js。
- 已购买域名解析到 Vercel。
- 可使用 Cloudflare 管理 DNS、CDN、安全规则和缓存。
- 主要面向海外访问，不按中国大陆备案部署设计。

## 9. Assumptions

- 网站主要部署海外。
- 一期优先快速上线和低维护成本。
- 产品数量会持续增加。
- 未来需要运营人员通过后台维护产品。
- 暂不做在线报价、在线交易、支付和会员系统。
- 第一版重点是公司介绍、产品介绍、产品对比、信任背书和询盘转化。
